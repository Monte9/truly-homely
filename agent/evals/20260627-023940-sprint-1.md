# VERDICT — Sprint 1: Menu browsing experience

Date: 2026-06-27 (UTC)
Build: production (`pnpm build` + `pnpm start -p 3310`)
Driver: Playwright/Chromium headless (`executablePath: /opt/pw-browsers/chromium`)
Data source: `src/data/menu.json` (19 items, 6 categories, 31 photos)

## Overall verdict: FAIL

One hard gate fails: a reproducible `/favicon.ico` 404 logs a console error on
**every** page (index and dish detail) on first visit. All other hard gates and
all six acceptance criteria pass with interaction evidence. The failure is
narrow and cosmetic but the gate is binary ("zero console errors", and AC #6
says "no console errors"), so the sprint does not pass as-is.

---

## Hard gates

1. **`pnpm build` exits 0 — PASS.** Build completed: "Compiled successfully",
   23 static pages generated, prebuild emitted "menu: 19 dishes, 31 photos, 6
   categories". Exit code 0.

2. **Zero console errors — FAIL.** Every page logs exactly one console error on a
   fresh browser: `Failed to load resource: the server responded with a status
   of 404 (Not Found)`. Source identified as `/favicon.ico` (curl
   `http://localhost:3310/favicon.ico` -> 404; no `icon`/`favicon` file in
   `src/app/` or `public/`). Reproduced 3/3 fresh-browser loads on `/` and 1/1
   on `/dish/khaman-dhokla`. (Within a single browser the error is hidden on
   later loads by the browser's negative cache, but it occurs on every visitor's
   first load.) No other console errors observed anywhere.

3. **No broken images — PASS.** After scrolling the full index and waiting for
   image completion, all 19 grid images have `naturalWidth > 0` (brokenCount 0).
   Dish detail (Khaman Dhokla) had 4 `<img>` (1 hero + 3 thumbs) all with
   `naturalWidth > 0`, 0 broken. NOTE: an initial pre-scroll snapshot reported 4
   below-the-fold cards (laddo, momos, pakora, bhajji) with `naturalWidth 0`;
   this was a lazy-load/decode timing artifact — the underlying raw files and the
   `/_next/image` optimizer return 200 for all of them, and they load once in
   view. Not a real failure.
   Evidence: `agent/evals/shots/index-1280.png`, `dish-khaman-1280.png`.

4. **No horizontal overflow at 390px — PASS.** Index `scrollWidth 390 == innerWidth 390`;
   dish detail `scrollWidth 390 == 390`. Evidence: `agent/evals/shots/index-390.png`,
   `dish-khaman-390.png`.

5. **All ACs demonstrated by interaction — PASS** (see AC section; favicon issue
   is gate 2, not a missing AC).

6. **No regressions at 1280px and 390px — PASS.** Index and dish detail render
   correctly at both widths; `scrollWidth == innerWidth` at 1280 and 390 on both
   pages. Evidence: the four index/detail screenshots above.

7. **Menu reflects the data — PASS.** Grid `data-count="19"`, 19 unique-slug
   cards. Category controls exactly match `categories` in JSON (All + Breakfast,
   Snack, Dessert, Dinner, Lunch, Meal). Dessert filter -> exactly
   [shrikhand, monthal] (data has 2 dessert items); Meal -> exactly
   [bhindi-kadi-roti] (data has 1). Only Gujarati Muthiya carries a price
   (₹100.00) and it is the only card showing a price. No hardcoded/dropped items.

## Soft notes

- Strong, appetizing composition: warm cream palette, serif display heading,
  rounded cards with photo-on-top / name-below, pill filter with clear active
  state, "Navarathri Stall" badge. Reads like a real menu you want to scroll.
- Typography hierarchy is clear; price pill is legible; no near-invisible text.
- Photos are uniformly framed (object-cover) with no squish; detail gallery with
  active-thumb ring is a nice touch.
- Mobile (390px) collapses to a clean single column; filter bar scrolls
  horizontally within the viewport without causing document overflow.
- Motion is tasteful (entrance, hover lift, filter slide) and correctly disabled
  under reduced-motion.
- AC strengthening I would suggest for future sprints: AC #6 already covers "no
  console errors" — recommend explicitly listing favicon/app-icon presence as
  part of the no-404 expectation so this class of failure is caught at align time.

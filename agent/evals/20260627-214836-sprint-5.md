# Sprint 5 - Add to cart + checkout - Verdict

Goal: add-to-cart and a checkout page. Evaluator-aligned ACs (alignment note
this session): numeric INR prices everywhere, cart count = sum of quantities,
quantity bounds, editable cart, dual empty states (cart + checkout), correct
grand total, "Place Order" -> native "Coming soon" alert + clear, persistence,
mobile + a11y, stable testids.

## Implementation

- **Prices:** all 19 dishes priced (numeric, whole rupees) in the source CSV;
  `build-menu.mjs` now emits numeric `price`; `formatINR` (Intl en-IN, ₹, no
  decimals) used on cards, cart, and checkout. No `₹100.00`-style strings.
- **Cart state:** `CartProvider` (React context) persisted to `localStorage`
  (`th.cart.v1`); count = sum of quantities; quantities never go below 1
  (decrement at 1 removes the line); remove deletes a line.
- **Add to cart:** each card has an Add-to-cart button that becomes a − qty +
  stepper once in the cart.
- **Cart drawer:** floating cart button (bottom-right, count badge) opens a
  slide-in drawer with line photo/name/unit price/qty/line total, remove,
  subtotal, and a Checkout link; warm empty state.
- **Checkout (`/checkout`):** itemized summary + grand total (`cart-total`);
  "Place Order" fires `window.alert("Coming soon")` then clears the cart; empty
  state shown when the cart is empty (no placeable order).

## Verification (Playwright, prod build on :3070)

- Price shows ₹ with no `₹\d+\.\d{2}` strings.
- add -> count 1; increment -> 2; add 2nd dish -> 3; **persists 3 after reload**.
- Drawer shows 2 lines, subtotal ₹290; checkout shows 2 items, total **₹290**
  (80×2 + 130) — matches.
- Place Order: native alert text exactly **"Coming soon"**; afterwards checkout
  shows empty state and the cart count indicator is gone (cart cleared).
- No horizontal overflow at 390px on menu and checkout; **0 console errors**;
  `pnpm build` exits 0.

## Overall verdict: PASS

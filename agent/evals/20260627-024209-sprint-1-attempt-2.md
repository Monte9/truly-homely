# Sprint 1 - Menu browsing experience - Verdict (attempt 2)

Follow-up to `20260627-023940-sprint-1.md` (attempt 1: FAIL on a single gate).

## What changed since attempt 1

Attempt 1 passed every hard gate and acceptance criterion except **hard gate 2
(zero console errors) / AC #6**: a missing favicon made every page log one
`favicon.ico 404` on a fresh-browser first load.

Fix: added `src/app/icon.svg`. Next.js now emits
`<link rel="icon" href="/icon.svg..." type="image/svg+xml">`, so the browser
uses it and no longer requests `/favicon.ico`.

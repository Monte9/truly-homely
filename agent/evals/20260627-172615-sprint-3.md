# Sprint 3 - Card carousel + fullscreen lightbox + tags - Verdict

Direct requests from Monte (2026-06-27):
1. No per-dish detail page; clicking a photo opens it full screen with an X to close.
2. A swiper with dots on each card to show/swipe multiple images.
3. Show all tags on the card itself.

## Changes

- Removed the `/dish/[slug]` route and `DishGallery` (no detail page).
- `DishCard` is now a swipeable image carousel (scroll-snap, touch + desktop
  hover arrows) with dot indicators shown only when a dish has >1 photo.
  Tapping any photo opens the lightbox at that image.
- New `Lightbox`: fullscreen overlay, object-contain image, X to close, Esc /
  backdrop-click to close, prev/next arrows + dots + arrow keys to move through
  a dish's photos. Body scroll locked while open.
- All of a dish's `types` and `tags` now render as chips in the card body.

## Verification (Playwright, preinstalled Chromium, prod build on :3050)

- 19 cards render; `a[href^="/dish/"]` count = **0** (detail route gone).
- Khaman Dhokla (3 photos) shows **3 carousel dots**; card text includes
  "Breakfast", "Snack", and "Navarathri Stall" (all types + tags).
- Tapping a photo opens `[data-testid="lightbox"]` (visible) with 3 nav dots;
  the X (`[data-testid="lightbox-close"]`) closes it (count back to 0).
- Single-photo cards (e.g. Gujarati Muthiya) show no dots and the price ₹100.00.
- **0 console errors**; checked at 1280px and 390px. `pnpm build` exits 0.

## Overall verdict: PASS

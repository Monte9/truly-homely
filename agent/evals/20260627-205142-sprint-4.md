# Sprint 4 - Lightbox swipe/z-index + smoother motion - Verdict

Direct requests from Monte (2026-06-27):
1. In the fullscreen photo view, swiping does nothing; you must press the arrows.
2. The lightbox buttons sit behind the photo (need z-index).
3. Nicer open/close animation for the fullscreen photo.
4. Category filtering animates cards "from the top" and feels jarring; make it a
   smooth fade.

## Changes

- **Lightbox swipe:** the active photo is a `drag="x"` motion element; an
  offset+velocity threshold pages to the next/previous photo. Touch and mouse
  drag both work; arrow keys and buttons still work.
- **Z-index:** close, prev/next, and dots are now `z-30` (image stage `z-10`),
  so controls always sit above the photo. Added subtle blur/shadow for contrast.
- **Open/close motion:** backdrop fades; the image stage scales from 0.9 -> 1 on
  open and 1 -> 0.92 on close; photo-to-photo uses a directional slide+fade.
- **Filtering:** dish cards now fade + gently scale in (removed the vertical
  top-slide), with a smaller stagger, so changing category reads as a smooth
  crossfade instead of items dropping in from above.
- All motion respects `prefers-reduced-motion`.

## Verification (Playwright, preinstalled Chromium, prod build, 390px touch)

- Open lightbox, drag left -> active photo index 0 -> 1 (swipe works).
- Next-arrow hit-test returns "arrow-on-top" (controls above the photo).
- Close button dismisses the lightbox; 0 console errors.
- `pnpm build` exits 0.

## Overall verdict: PASS
